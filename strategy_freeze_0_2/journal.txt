CONTEXT
-------
Strategy v0.2 (strategy_freeze_0_2/) is a walk-forward trend-following system
on WIG20TR using MA crossover and/or momentum filter, with breakout entry,
trailing + absolute stop exits, and Monte Carlo / block bootstrap robustness
checks.  This session wired it up as a daily automated tool.

WORK COMPLETED
--------------

1. OUTPUT DESIGN
   Agreed output taxonomy for daily operational use:
   - Tier 1 (daily): signal state (IN/OUT), action (ENTER/EXIT/HOLD), stop
     levels, MA and momentum filter state
   - Tier 2 (on signal change): OOS performance metrics
   - Tier 3 (weekly/on-demand): charts

2. NEW FILE: daily_output.py  (strategy_freeze_0_2/)
   Module called at end of runfile. Produces four artefacts into outputs/:
     signal_status.txt   — human-readable status block for email body
     signal_log.csv      — appended daily history (one row per run)
     equity_chart.png    — 3-panel chart: equity, drawdown, MA filter ratio
     signal_snapshot.json — machine-readable full snapshot for GH Actions + Sheets

   Key design decisions:
   - CARRY record in wf_trades is single source of truth for open position state
   - Stop levels: both trailing (peak × (1-X)) and absolute (entry × (1-stop_loss))
     reported; binding stop = max of the two
   - MA filter state recomputed from raw price series using active window params
   - Momentum filter state recomputed using price.shift(21)/price.shift(252)-1
     (exact replica of compute_momentum() in strategy_test_library)
   - active_filter_on reflects whichever filter mode is live (ma or mom)
   - action key written to snapshot JSON (not underscore-prefixed) for GH Actions
   - All file writes are atomic (write to .tmp then rename)

   Runfile integration — add at end of strat_test_runfile_with_MC.py:
     from daily_output import build_daily_outputs
     build_daily_outputs(
         wf_equity=wf_equity, wf_trades=wf_trades,
         wf_metrics=wf_metrics, wf_results=wf_results,
         bh_equity=bh_equity, bh_metrics=bh_metrics,
         df=df, output_dir="outputs", price_col="Zamkniecie",
     )

3. NEW FILE: .github/workflows/daily_strategy.yml
   GitHub Actions workflow:
   - Schedule: 18:00 UTC (winter) + 17:00 UTC (summer) on weekdays,
     covering 19:00 Warsaw in both DST states
   - Steps: checkout → python 3.11 → pip install → run runfile →
     parse snapshot → Google Drive upload → conditional email → GH artefact
   - Email sent ONLY when action != HOLD (i.e. ENTER or EXIT)
   - workflow_dispatch with force_notify=true overrides for testing
   - Credentials: raw JSON secret GOOGLE_CREDENTIALS written to
     /tmp/credentials.json (matching existing pattern in repo)
   - Parse step uses absolute path (${{ github.workspace }}/...) and
     eval "$(python ...)" pattern to avoid heredoc subshell quoting issues
   - Step 9 (verify outputs) fails loudly if snapshot not written

   Required secrets:
     GOOGLE_CREDENTIALS      raw service account JSON
     GDRIVE_FOLDER_ID        Drive folder ID
     NOTIFY_EMAIL_TO         alert recipient
     GMAIL_USER              sender Gmail address
     GMAIL_APP_PASSWORD      Gmail App Password

6. GOOGLE SHEETS INTEGRATION: Code.gs
   Two-function Apps Script for a new workbook:
     importSignalLogRow()  — appends new rows from signal_log.csv daily,
                             deduplicates by date, applies conditional
                             formatting (green ENTER, red EXIT, yellow HOLD)
     refreshDashboard()    — rebuilds Dashboard tab from signal_snapshot.json,
                             coloured cells for signal/action/filter state,
                             stop gap turns red < 5%, green/red for all metrics

   Trigger: setupTrigger() registers daily at 20:00 local time.
   Test functions: testImportLog(), testDashboard(), testRefreshAll()

   Dashboard sections:
     SIGNAL — signal (IN/OUT), action, price
     OPEN POSITION — entry date/price, days, unrealised, peak price
     STOP LEVELS — trailing, absolute, binding stop, gap to stop
     FILTERS [active: MA/MOM] — active filter state, MA cross rows
       (fast/slow/gap), Momentum rows (12m-1m value); active filter
       labelled (ACTIVE) in bold, inactive in grey
     OOS PERFORMANCE — CAGR/Sharpe/MaxDD/CalMAR vs B&H in 3-column layout
     ACTIVE WINDOW & PARAMETERS — window dates, X/Y/fast/slow/stop_loss/mode
